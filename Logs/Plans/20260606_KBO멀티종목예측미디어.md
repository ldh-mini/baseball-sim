# Scoracle — KBO 멀티 종목 예측 미디어 사이트 플랜

작성일: 2026-06-06
상태: 초안

## Context

pickswise.com/mlb/picks 구조를 벤치마크하여, **베팅/affiliate 요소를 제거하고 자체 모델 승부예측 중심의 합법 예측 미디어**로 한국 로컬 서비스를 구축한다.

### pickswise 핵심 구조 (벤치마크)
- URL: `/{종목}/{콘텐츠타입}/` 격자형 (종목 × 콘텐츠 매트릭스)
- 종목(NFL/NBA/MLB…)마다 동일 콘텐츠 세트(picks/predictions/odds/news) 반복 → 횡복제 구조
- 핵심 컴포넌트: **예측 카드** (매치업·전문가·픽타입·신뢰도 별점·배당·CTA)
- 수익 모델: 무료 콘텐츠 + 북메이커 affiliate 퍼널

### 현재 Scoracle 상태 (출발점)
- 단일 모놀리식 `kbo-simulation.jsx` (2171줄), **라우터 없음** — `useState("virtual")` 탭 전환만
- 탭 4종: 가상 대결(VirtualTab) / 오늘의 경기(TodayTab) / 누적 적중률(PredictionLogTab) / 백테스트(BacktestTab)
- **야구 전용** — KBO 10개 구단 하드코딩, 종목 추상화 없음
- 예측 엔진·시뮬레이션·백테스트·적중률 로깅 인프라는 이미 보유 (v9.6)
- 스택: React 19 + Vite 8 + Tailwind 3, 빌드 정적 배포(PWA)

### 목표 상태
1. `/{종목}/{콘텐츠}/` 격자형 라우팅으로 전환 (KBO→K리그→KBL 횡전개)
2. pickswise식 **예측 카드 그리드** — 배당 자리에 자체 모델 승리확률%·예상스코어·신뢰도·AI 요인분석 해설
3. 종목 추상화 레이어 — 새 종목 추가 시 데이터 어댑터만 구현하면 동일 UI 재사용
4. SEO 롱폼 콘텐츠/뉴스 슬롯 (유입 퍼널)

### 컴포넌트 매핑 (베팅 → 합법 예측)
| pickswise | Scoracle |
|---|---|
| 북메이커 배당 비교 | 자체 모델 승리확률 % |
| 별점 신뢰도 2★~5★ | 예측 신뢰도 (유지) |
| sportsbook 가입 버튼 | "상세 분석 / 시뮬레이션 돌리기" CTA |
| Money Line / Run Line 픽 | 승패 예측 / 예상 스코어 |
| 전문가 분석 1~3문장 | AI 요인 분석 해설 (v9.6 자산) |
| affiliate 프로모 배너 | 제거 (구독/광고로 대체) |

## 리스크 / 주의사항

- **법적 리스크 (최우선)**: 배당·affiliate·북메이커 링크는 일절 포함 금지. "예측/분석 미디어" 포지셔닝 유지. 사설 베팅 유도로 해석될 표현(픽 적중 보장, 수익률 강조 등) 회피.
- **모놀리스 분해 리스크**: 2171줄 단일 파일 분해 시 기존 예측 정확도/백테스트 결과가 깨지지 않도록 로직은 "이동"만 하고 알고리즘은 손대지 않는다. 분해 전후 동일 입력→동일 예측 회귀 확인.
- **데이터 소스**: 현재 정적 JSON(빌드 타임 크롤)에 의존. 멀티종목 확장 시 종목별 크롤러/데이터 파이프라인 필요 — 이번 플랜 범위는 KBO만 실데이터, 타 종목은 스텁.
- **한글 URL**: `/kbo/예측/` 한글 경로는 인코딩/SEO 이슈 가능. 영문 슬러그(`/kbo/predictions/`) + 한글 라벨 표시 방식도 검토.
- **빌드 라우팅**: 정적 배포 시 딥링크 404 방지 위해 SPA fallback 또는 프리렌더 설정 필요.
