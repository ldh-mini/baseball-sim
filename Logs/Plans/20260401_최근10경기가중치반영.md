# 최근 10경기 데이터 시뮬레이션 추가가중치 반영 플랜

작성일: 2026-04-01
상태: 1~5단계 완료 (57.1% 적중률 달성)

## Context

현재 시뮬레이션은 **시즌 누적 스탯**만 사용하여 예측한다. `recentForm` 필드가 존재하지만 시즌 OPS 기반의 정적 값이며, 실제 최근 경기 흐름을 반영하지 못한다.

야구에서 선수의 최근 10경기 성적은 컨디션·폼·상대 투수 궁합 등 단기 트렌드를 포착하는 핵심 지표다. 이를 반영하면:
- 핫스트릭/슬럼프 선수의 가중치를 동적으로 조절 가능
- 시즌 초반 소표본 문제를 최근 성적으로 보완
- 부상 복귀, 타순 변경 등 최신 상황 반영

**현재**: 시즌 전체 AVG/OBP/SLG → `recentForm` 정적 값 (0.92~1.08)
**목표**: 최근 10경기 개인 스탯 크롤링 → 동적 `recentForm` + 보조 가중치 반영

## 데이터 소스 분석

### KBO 사이트 크롤링 가능 페이지

| 페이지 | URL 패턴 | 데이터 |
|--------|---------|--------|
| 타자 일별 기록 | `/Record/Player/HitterBasic/GameLog.aspx` | 날짜별 H/AB/BB/K/HR/RBI |
| 투수 일별 기록 | `/Record/Player/PitcherBasic/GameLog.aspx` | 날짜별 IP/H/R/ER/BB/K |
| 경기 박스스코어 | `/Schedule/GameCenter/Main.aspx?gameDate=YYYYMMDD` | 전체 박스스코어 |

> **주의**: KBO 사이트는 ASP.NET PostBack 방식. 시즌 변경처럼 Playwright가 필요할 수 있음 (crawl-2025.mjs 선례).
> **대안**: Statiz(statiz.co.kr) 사이트는 정적 HTML이라 GET으로 크롤링 가능할 수 있음.

### 크롤링 전략: 두 가지 옵션

**옵션 A — 팀 단위 최근경기 (권장)**
- KBO `/Record/Player/HitterBasic/Basic1.aspx`에서 이미 시즌 누적 크롤링 중
- 최근 10경기 전용 필터가 있으면 사용, 없으면 **시즌 누적 - (시즌-10경기 누적)** 차분 방식
- 장점: 기존 크롤링 인프라 재사용, 추가 API 최소
- 단점: 정확한 게임 로그가 아닌 근사치

**옵션 B — 개인 게임로그 크롤링**
- 각 선수별 게임로그 페이지 크롤링
- 장점: 정확한 최근 10경기 데이터
- 단점: 선수 수 × 개별 페이지 → 크롤링 시간 ↑ (100명+ × 2초 = 3분+)

**옵션 C — 하이브리드 (최종 권장)**
- 주전급 선수(팀당 라인업 9명 + 선발투수 5명 = 14명 × 10팀 = 140명)만 크롤링
- Playwright 사용, 병렬 처리로 2~3분 내 완료
- 캐시 파일(`recent-stats.json`)에 저장, 당일 중 재실행 시 캐시 사용

### 3단계: `blend-stats.mjs` 3차 블렌딩 추가

- [x] 기존 2-layer 블렌딩 위에 3번째 레이어 추가 완료
- [x] 블렌딩 공식 설계:
```
최종 스탯 = blended_season × (1 - recentWeight) + recent10 × recentWeight

recentWeight = min(0.30, recentGames / 10 × 0.30)
```
  - 최근 10경기 데이터가 충분하면 최대 30% 가중치
  - 10경기 미만이면 비례 감소 (5경기 → 15%)
  - 0경기 → recentWeight=0 (기존 블렌딩 그대로)
- [ ] 투수도 동일 구조:
```
최종 ERA = blended_era × (1 - rW) + recent_era × rW
최종 WHIP = blended_whip × (1 - rW) + recent_whip × rW
```
- [ ] 로그 출력에 최근 10경기 반영 여부 표시

### 4단계: `recentForm` 동적 계산 로직

- [ ] `kbo-simulation.jsx`에서 정적 `recentForm` 값 대신 동적 계산 함수 추가:
```javascript
function calcRecentForm(seasonStats, recent10Stats) {
  if (!recent10Stats || recent10Stats.games < 3) return 1.0;
  // 타자: 최근10경기 OPS vs 시즌 OPS 비교
  const seasonOPS = seasonStats.obp + seasonStats.slg;
  const recentOPS = recent10Stats.obp + recent10Stats.slg;
  const ratio = recentOPS / Math.max(seasonOPS, 0.500);
  return _.clamp(ratio, 0.85, 1.15);
}
```
- [ ] `prob()` 메서드에서 기존 `bF = _.clamp(b.recentForm, 0.92, 1.08)` 유지
  - 동적 계산된 값이 이미 0.85~1.15 범위이므로, 최종 클램프에서 0.92~1.08로 제한
  - **핫스트릭 선수**: 최근 OPS 0.900 vs 시즌 0.700 → ratio 1.286 → clamp 1.15 → 최종 1.08
  - **슬럼프 선수**: 최근 OPS 0.500 vs 시즌 0.750 → ratio 0.667 → clamp 0.85 → 최종 0.92
- [ ] 투수 `recentForm`도 동일하게:
```javascript
function calcPitcherRecentForm(seasonStats, recent10Stats) {
  if (!recent10Stats || recent10Stats.games < 2) return 1.0;
  // ERA 역수 비율 (ERA 낮을수록 좋으므로 역수)
  const seasonERA = Math.max(seasonStats.era, 1.0);
  const recentERA = Math.max(recent10Stats.era, 1.0);
  const ratio = seasonERA / recentERA; // 최근 ERA < 시즌 → ratio > 1 → 좋은 폼
  return _.clamp(ratio, 0.85, 1.15);
}
```

## 예상 가중치 체계 (전체 그림)

```
최종 스탯 = f(2025 prior, 2026 시즌, 최근 10경기)

Layer 1: 베이지안 블렌딩
  blended = 2026_real × w₁ + 2025_prior × (1-w₁)
  w₁ = PA / (PA + REG_PA)   [0 → 2025 100%, PA=120 → 50:50]

Layer 2: 최근 10경기 오버레이
  final = blended × (1-w₂) + recent10 × w₂
  w₂ = min(0.30, games/10 × 0.30)   [0 → 블렌딩 100%, 10경기 → 70:30]

Layer 3: recentForm 승수 (prob() 내)
  bF = clamp(recentOPS / seasonOPS, 0.92, 1.08)
  → 최종 확률에 ±8% 추가 조절
```

이 3-layer 구조로:
- **시즌 초반**: 2025 prior가 지배적 → 안정적 예측
- **시즌 중반**: 2026 시즌 데이터가 주도, 최근 10경기로 트렌드 포착
- **핫스트릭/슬럼프**: recentForm 승수가 단기 변동 반영
