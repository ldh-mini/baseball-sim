# 팀 레이팅 동적 업데이트 + 선발투수 자동 감지 플랜

작성일: 2026-04-06
완료일: 2026-04-07
상태: **구현 완료** (7/7 단계, 후속 과제 식별)

## Context

### 현재 문제
**v9.0 실전 예측(4/1~4/5) 분석 결과**, 적중률 60%(9/15)의 주요 실패 원인 2가지가 확인됨:

1. **팀 레이팅 정적 고정**
   - `KBO_TEAMS[team].teamRating` 값이 2025 시즌 기준으로 하드코딩
   - 예: KT=76, 한화=75 — 그러나 2026 4/1~4/2 KT는 **개막 5연승**, 경기당 10점 이상 득점
   - `getOddsMod()` / Elo 계산에 2025 레이팅 반영 → 실제 전력과 괴리
   - 4/2 KT@한화 예측(한화 58%) → 실제 KT 13-8 승, 롯데 6연패/SSG 4연승 등 모멘텀 미반영

2. **선발투수 수동 등록 부담**
   - 매일 경기 전 선발투수 10명을 수동으로 `sim-today.mjs`에 입력
   - 미등록 시 starters[0]로 폴백 → 4/1 8명, 4/2 10명 매칭 오류 경험
   - `findSP()`는 이름 부분일치에 의존 — 신인/트레이드 선수 누락 위험
   - 4/2 추정치 선발 매칭 적중률 **25% (1/4)** vs 정규 선발 **73% (8/11)**

### 목표 상태
- **팀 레이팅**: 2026 시즌 진행 중 전적(승/패/득실점) 기반 자동 갱신. 시즌 초반(n<20)은 2025 prior와 블렌딩
- **선발투수**: KBO 공식 사이트에서 당일 예고선발 자동 크롤링 → 팀 starters에 없으면 자동 등록 (최근 10경기 ERA/WHIP 기반 스탯 추정)

### 성공 지표
- `sim-today.mjs` 실행 시 수동 편집 없이 당일 경기 예측 완료
- 실전 적중률 **60% → 65%+** (특히 추정치 선발/모멘텀 경기 개선)

## 구현 단계

### 1단계: 팀 순위/전적 크롤러 (`crawl-teamrank.mjs`)

- [x] KBO 공식 팀 순위 페이지 URL 분석
  - `https://www.koreabaseball.com/Record/TeamRank/TeamRank.aspx` (또는 일자별)
  - cheerio 로 팀별 승/패/무, 득점(RS), 실점(RA) 파싱
- [x] 출력 포맷 정의
  ```json
  {
    "crawlDate": "2026-04-06",
    "teams": {
      "kt": { "w": 5, "l": 0, "t": 0, "rs": 52, "ra": 38, "pct": 1.000 },
      "hanwha": { "w": 2, "l": 4, "t": 0, "rs": 35, "ra": 48, "pct": 0.333 },
      ...
    }
  }
  ```
- [x] `team-stats.json` 파일로 저장 (일별 갱신)
- [x] 팀명 → 내부 ID 매핑 (`NM` 상수 재사용)
- [x] 테스트: 10개 팀 전원 크롤링 검증

### 2단계: 팀 레이팅 블렌딩 로직 (`blend-stats.mjs` 확장)

- [x] `team-stats.json` 로드 (있으면)
- [x] 2026 전적 기반 동적 `teamRating` 계산
  ```
  rating_2026 = 50 + (pct - 0.5) × 80 + runDiff_per_game × 3
  // pct=1.000, runDiff=+3 → 50 + 40 + 9 = 99 (KT 현재 수준)
  // pct=0.333, runDiff=-2 → 50 - 13.3 - 6 = 30.7 (한화 현재 수준)
  ```
- [x] 시즌 초반 블렌딩 (베이지안 회귀)
  ```
  w = min(1.0, games_played / 30)  // 30경기 기준 완전 전환
  final_rating = w × rating_2026 + (1-w) × rating_2025
  ```
- [x] `record` 필드도 동시 갱신 (`w`, `l`, `t`, `rs`, `ra`)
- [x] jsx 자동 반영 (기존 블렌딩과 동일한 regex replace 패턴)
- [x] 로그 출력: 팀별 `기존rating → 신rating (w경기, pct)`

### 3단계: 당일 일정 + 예고선발 크롤러 (`crawl-schedule.mjs`)

## 리스크 / 주의사항

### 1. 팀 레이팅 과적합
- 시즌 초반(5~10경기)은 소규모 샘플 → 극단값 발생 가능
- **대응**: 베이지안 회귀 가중치 `w = min(1.0, games/30)` 적용, 초반엔 prior 우세
- **대응**: rating 상한/하한 클램프 (`_.clamp(rating, 40, 100)`)

### 2. KBO 사이트 구조 변경
- ASP.NET PostBack 페이지 — HTML 구조 변경 시 파싱 깨질 위험
- **대응**: 폴백 데이터 소스 최소 1개 유지 (네이트/다음 스포츠)
- **대응**: 파싱 실패 시 이전 캐시 사용 + 경고

### 3. 예고선발 변경
- 경기 당일 선발 교체 빈번 (부상, 우천)
- **대응**: 경기 시작 2~3시간 전 재크롤링 권장
- **대응**: 수동 오버라이드 옵션 유지 (CLI argv로 선발 지정)

### 4. 누락 선발 스탯 추정 정확도
- 최근 10경기에 없는 투수(신인, 2군 콜업)는 팀 bullpen 평균 사용 → 부정확
- **대응**: 2025 legacy 데이터 백업 조회 추가
- **대응**: `fip` 필드 누락 시 `calcFIP()` 재계산

### 5. 모멘텀/연승연패 반영
- 단순 팀 레이팅 블렌딩만으로는 "최근 5연승 기세"를 완전히 포착하지 못함
- **대응**: Layer 2C 추가 — 최근 10경기 팀 승률 기반 `teamForm` 보정 (±5%)
- **확장**: 향후 Phase 2.6에서 모멘텀 모델링 강화
