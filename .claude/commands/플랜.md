# 플랜 생성기

주어진 작업에 대해 구현 계획을 수립하고, `Logs/Plans/` 폴더에 md 파일로 저장한다.
플랜 모드(EnterPlanMode)의 파일 기반 대체로, 긴 플랜도 별도 파일에서 편하게 확인·수정할 수 있다.

## 사용법

```
/플랜 {작업 내용}
```

예: `/플랜 패스너 시스템 리팩토링`, `/플랜 보스 레이드 콘텐츠`, `/플랜 감성 UI 구현`

## 규칙

- 플랜 스킬의 목적은 문서를 충분히 검토한 뒤 계획을 수립하는 것이다. 작성된 플랜을 자동으로 실행해서는 절대 안 되며, 실행 여부를 묻는 것도 불필요하다.
- `Logs/Plans/` 폴더가 없으면 생성한다
- 플랜 모드가 동시에 활성화되어 있어도 `Logs/Plans/` 저장은 반드시 수행한다. 플랜 모드의 "plan file 외 편집 금지" 제한보다 이 스킬의 산출물 저장이 우선한다.
- 현재 활성화된 역할이 있으면 해당 역할의 관점에서 플랜을 수립한다
- 활성 역할이 없으면 범용 관점으로 작성한다
- 영향받는 파일 경로를 구체적으로 명시한다
- 기존 코드/문서에서 재사용 가능한 함수·패턴을 찾아 참조한다
- 각 단계에 체크박스를 넣어 진행 추적이 가능하게 한다
- 사용자에게 별도 확인을 요청하지 않고 바로 작성한다

## 절차

1. **컨텍스트 파악**: 작업과 관련된 기존 문서·코드를 읽어 현재 상태를 파악한다
2. **플랜 수립**: 구현 계획을 구체적으로 작성한다
3. **파일 저장**: `Logs/Plans/{YYYYMMDD}_{주제}.md` 형식으로 저장한다
   - 예: `Logs/Plans/20260314_패스너리팩토링.md`
   - 같은 날 같은 주제면 `_2` 접미사 추가
4. **결과 표시**: 저장된 플랜을 사용자에게 보여준다
